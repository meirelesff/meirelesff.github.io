Regressão linear múltipla

Introdução

Estas notas tem um objetivo bem simples: ensinar como usar o pacote fixest para rodar regressões lineares com efeitos fixos (além de outras utilidades).

Usaremos como exemplo, novamente, os dados de Ross (2001). Vamos carregá-los com:

library(haven)
ross <- read_dta("doesoil_clean.dta")

Fixest

O fixest serve basicamente para rodar modelos de regressão linear e generalizados de forma rápida – especialmente em bancos grandes, com muitos grupos e ou especificações complexas. O primeiro passo para usá-lo, como sempre, é instalar e carregar o pacote:

install.packages("fixest")
library(fixest)

Isso feito, vamos rodar um modelo simples para regredir regime em oil com a função feols, como viemos fazendo:

feols(regime ~ oil, data = ross)
NOTE: 1,822 observations removed because of NA values (LHS: 19, RHS: 1,806).
OLS estimation, Dep. Var.: regime
Observations: 2,708 
Standard-errors: IID 
             Estimate Std. Error  t value  Pr(>|t|)    
(Intercept)  2.403693   0.150973  15.9214 < 2.2e-16 ***
oil         -0.172361   0.010298 -16.7366 < 2.2e-16 ***
---
Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1
RMSE: 7.30456   Adj. R2: 0.09347

O resultado é o mesmo que obteríamos com lm:

lm(regime ~ oil, data = ross)

Call:
lm(formula = regime ~ oil, data = ross)

Coefficients:
(Intercept)          oil  
     2.4037      -0.1724

O grande diferencial do fixest, no entanto, é podermos usar uma sintaxe simples para declarar efeitos fixos (within), algo que pode ser feito com:

feols(regime ~ oil | cty_name, data = ross)
NOTE: 1,822 observations removed because of NA values (LHS: 19, RHS: 1,806).
OLS estimation, Dep. Var.: regime
Observations: 2,708 
Fixed-effects: cty_name: 130
Standard-errors: Clustered (cty_name) 
    Estimate Std. Error  t value Pr(>|t|) 
oil 0.000353   0.011302 0.031206  0.97515 
---
Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1
RMSE: 3.76654     Adj. R2: 0.746899
                Within R2: 2.544e-7

No código acima, variáveis que aparecem depois do | são entendidas como efeitos fixos, isto é, tratadas como variáveis discretas e que, nesse caso, ocupam o lugar do intercept, indicando a média de regime para cada país quando oil é zero.

Outra utilidade do fixest é permitir incluirmos erros-padrão por cluster – para sinalizar que nossas observações dentro de um mesmo país não são iid. Fazemos isso com:

feols(regime ~ oil | cty_name, data = ross, cluster = ~ cty_name)
NOTE: 1,822 observations removed because of NA values (LHS: 19, RHS: 1,806).
OLS estimation, Dep. Var.: regime
Observations: 2,708 
Fixed-effects: cty_name: 130
Standard-errors: Clustered (cty_name) 
    Estimate Std. Error  t value Pr(>|t|) 
oil 0.000353   0.011302 0.031206  0.97515 
---
Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1
RMSE: 3.76654     Adj. R2: 0.746899
                Within R2: 2.544e-7

Note que, embora estimativas não tenham mudado, erros-padrão e p-valores foram alterados.

Tabelas

Outra utilidade do fixest é permitr a criação e exportação rápida de tabelas padronizadas com resultados de modelos de regressão. Para isso, bastar usarmos a função etable. A título de exemplo, vamos criar três modelos para compará-los em uma tabela:

m1 <- feols(regime ~ oil, data = ross)
NOTE: 1,822 observations removed because of NA values (LHS: 19, RHS: 1,806).
m2 <- feols(regime ~ oil | cty_name, data = ross)
NOTE: 1,822 observations removed because of NA values (LHS: 19, RHS: 1,806).
m3 <- feols(regime ~ oil | cty_name + year, data = ross)
NOTE: 1,822 observations removed because of NA values (LHS: 19, RHS: 1,806).

O primeiro é o nosso modelo simples de regime ~ oil, que é acrescido de efeitos fixos para países e, posteriormente, de efeitos fixos para anos. Para criar a tabela, basta usarmos:

etable(m1, m2, m3)
                                 m1              m2              m3
Dependent Var.:              regime          regime          regime
                                                                   
Constant          2.404*** (0.1510)                                
oil             -0.1724*** (0.0103) 0.0004 (0.0113) 0.0193 (0.0144)
Fixed-Effects:  ------------------- --------------- ---------------
cty_name                         No             Yes             Yes
year                             No              No             Yes
_______________ ___________________ _______________ _______________
S.E. type                       IID    by: cty_name    by: cty_name
Observations                  2,708           2,708           2,708
R2                          0.09380         0.75905         0.80002
Within R2                        --         2.54e-7         0.00086
---
Signif. codes: 0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1

Por padrão, as tabelas do fixest incluem uma série de informações que não usamos tanto, além de usar os nomes das variáveis brutas nas linhas. Abaixo, segue um exemplo de tabela mais compacta, com um padrão mais próximo do que costuma ser usado em artigos:

etable(m1, m2, m3, 
    digits = "r2", 
    fitstat = c("n", "g"), 
    se.below = T, 
    dict = c(regime = "Polity",
            oil = "Petróleo (% do PIB)",
            cty_name = "País",
            year = "Ano"),
    depvar = F, 
    #tex = T, style.tex = style.tex("aer"), # Para PDF!
    notes = "Notas: modelos de regressão estimados com o pacote fixest.",
    title = "Efeito do petróleo no regime")
                          m1       m2       m3
Constant             2.40***                  
                    (0.15)                    
Petróleo (% do PIB) -0.17***    0.00     0.02 
                    (0.01)     (0.01)   (0.01)
Fixed-Effects:      -------- -------- --------
País                     No      Yes      Yes
Ano                       No       No      Yes
___________________ ________ ________ ________
S.E. type                IID by: País by: País
Observations           2,708    2,708    2,708
G                      2,706      130      130
---
Signif. codes: 0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1

A maioria dos argumentos acima, como é possível ver, são intuitivos. De toda forma, para uma consulta mais detalhada de todas as possibilidades da função é possível obter ajuda com help(etable).1
